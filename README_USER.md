# Sales PVRP Scheduler - ръководство за потребителя

## Стартиране

Стартирайте приложението с двойно кликване върху `SalesPVRP.exe`.

Ако работите в development режим, командата е:

```bat
python run_gui.py
```

## Входен Excel файл

Използвайте Excel файл със следните задължителни колони:

- `client_id`
- `client_name`
- `sales_rep`
- `lat`
- `lon`
- `visit_frequency`

В папка `data` има примерни файлове:

- `input_clients_template.xlsx` - празен шаблон
- `sample_clients.xlsx` - примерни данни

## Избор и валидация на данни

1. В таб `Input` изберете Excel файла.
2. Изберете output папка.
3. Натиснете `Зареди данни`.
4. В таб `Validation` натиснете `Валидирай`.

Ако има редове със `ERROR`, коригирайте Excel файла и го заредете отново. Ако има само `WARNING`, оптимизацията може да продължи, но е добре да проверите предупрежденията.

## Стартиране на оптимизация

След успешна валидация отидете в таб `Run` и натиснете `Start`. Прогресът и текущите съобщения се виждат в лог панела.

## Отваряне на резултати

След успешен run отворете резултатите от таб `Results`:

- Excel график: `final_schedule.xlsx`
- HTML карта: `final_schedule_map.html`
- output папка с всички резултати

По подразбиране файловете са в избраната output папка. При стандартна release инсталация това е папка `output` до `SalesPVRP.exe`.

## Честоти на посещение

- `2` = два пъти месечно
- `4` = веднъж седмично
- `8` = два пъти седмично

## Ако OSRM не работи

Приложението може да работи и без OSRM. Ако OSRM не е наличен и fallback е включен, ще се използва приблизителна haversine матрица. Това е подходящо за тестове и диагностика, но за по-точни километри използвайте работещ OSRM server.

Проверете:

- дали OSRM server е стартиран;
- дали URL е правилен, например `http://localhost:5000`;
- дали `Fallback към haversine` е включен в настройките.

## Ако получите "No feasible solution"

Това означава, че при текущите правила и candidate routes не е намерен валиден график. Проверете таб `Logs` и Excel sheets `Validation` и `Candidate_Coverage`, ако са създадени.

Полезни действия:

- намалете `min_clients`;
- включете `allow_underfilled`;
- увеличете `candidates_per_rep`;
- увеличете `keep_top_n_per_rep`;
- увеличете `time_limit_seconds`;
- проверете фиксирани/забранени дни в Excel файла.

## Ако Excel файлът е отворен

Ако `final_schedule.xlsx` е отворен в Excel, приложението може да няма право да го презапише. Затворете файла и стартирайте оптимизацията отново или изберете друга output папка.

## Къде са output файловете

Обичайна структура:

```text
output/
  final_schedule.xlsx
  maps/
    final_schedule_map.html
  logs/
    run_YYYYMMDD_HHMMSS.log
```

## Къде са логовете

Логовете за run са в:

```text
output/logs/
```

Ако сте избрали друга output папка, логовете са в нейна подпапка `logs`.

## Чистене на cache

Можете безопасно да изтриете съдържанието на папка `cache`, когато искате следващият run да пресметне матриците и candidate routes отначало.

Не изтривайте `config.yaml` и файловете в `data`, освен ако знаете, че вече не са нужни.
